Os dados esto no formato .csv e, com eles em mos, o primeiro passo carreg-los em um data-frame, usando a funo pd.read_csv() do pandas.

Com os dados carregados, possvel averiguar que eles so constitudos por 142193 entradas, cada uma contando com 24 variveis distintas. possvel notar, tambm, que existem entradas para as quais nem todas as variveis esto instanciadas. Alm disso, nem todos os valores esto nos formatos que gostaramos que estivessem para serem processados.

Isso normal. Dados reais so cheios de falhas e problemas, e exigem trabalho e entendimento para serem utilizados da maneira correta. Por isso, necessrio realizar um pr-processamento para adequar os dados, antes de os passarmos para o modelo.

O primeiro passo escolher quais das variveis meteorolgicas nos interessam. No nosso caso, queremos prever se chover ou no amanh, ento RainTomorrow ser nossa varivel alvo. Para prev-la usaremos as variveis Rainfall, Humidity3pm, Pressure9am e RainToday, que sero nossas features.

Em seguida, iniciamos o pr-processamento, propriamente dito.

As variveis RainToday e RainTomorrow possuem dois valores possveis, Yes e No. Adeque esses valores, convertendo-os para $1$ e $0$, respectivamente.

A seguir, remova todas as entradas que no tenham instanciado os valores de todas as variveis de interesse com a funo dropna(), pois essas entradas so inteis para treinar nosso modelo.

Com os dados pr-processados, possvel, agora, plotar as distribuies das variveis de interesse para poder entender melhor como essas distribuies funcionam. Esse tipo de trabalho muito importante na implementao real de redes neurais, conhecer os dados fundamental para tirar o maior proveito do seu modelo e entender verdadeiramente seus resultados.

Dentre todas as distribuies das variveis de interesse, a que mais nos concerne a da varivel alvo, RainTomorrow, representada na figura acima.

Essa distribuio nos revela um grande desbalano entre os dois valores possveis dessa varivel, que constituiro as duas classes do nosso problema de classificao. Esse um dado importante, pois pode influenciar significativamente a capacidade preditiva do modelo treinado.

Existem maneiras de se lidar com o desbalanceamento dos dados, mas nesse tutorial utilizaremos os dados dessa forma. Isso significa que o baseline para a performance do nosso modelo deve ser $78\%$, isso porque, se um modelo chutasse que amanh no ir chover, todas as vezes, ele obteria uma performance dessa ordem e, como esperamos gerar um modelo mais "inteligente" que isso, esperamos tambm que a a nossa performance seja superior a essa.

def load_data(data_path='data/weatherAUS.csv')-> pd.DataFrame:
    """Funcao que importa dados de um arquivo csv, usando pandas"""
    # Insira seu cdigo aqui
    
    raw_data = pd.read_csv(data_path)

    return raw_data

def pre_processing(raw_data:pd.DataFrame)-> pd.DataFrame:
    """Funcao que filtra e limpa os dados meteorologicos para o treinamento"""
    # Insira seu cdigo aqui
    
    processed_data = raw_data.assign(RainToday = [1 if rain == "Yes" else 0 for rain in raw_data['RainToday']])
    processed_data = processed_data.assign(RainTomorrow = [1 if rain == "Yes" else 0 for rain in processed_data['RainTomorrow']])
    processed_data = processed_data.dropna()

    return processed_data


def split_data(data:pd.DataFrame, val_size= 0.2)-> np.array:
    """Funcao que separa seus dados em teste e treino conforme a proporcao val_size"""
    """Use a varivel global SEED como semente para a diviso aleatria"""
    # Insira seu cdigo aqui
    train, val = train_test_split(data, test_size=val_size, random_state=SEED)
    y_train = train["RainTomorrow"]
    y_val = val["RainTomorrow"]
    features = ["Rainfall", "Humidity3pm", "Pressure9am", "RainToday"]
    x_train = pd.DataFrame(train, columns=features)
    x_val = pd.DataFrame(val, columns=features)

    return x_train, x_val, y_train, y_val
